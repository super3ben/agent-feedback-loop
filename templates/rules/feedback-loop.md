# Feedback Loop Runtime Rule

Keep the installed synchronous surface prompt-only. Capture eligible dissatisfaction,
return the host's native response immediately, and allow the detached feedback reviewer
to publish immutable project Markdown under `.agent/reflections/`. Read that Markdown
only from a later matching prompt after the publication cutoff; do not store lesson
bodies in SQLite or introduce RAG.

Treat the convergence Probe as a bounded semantic adviser, not the feedback reviewer
or an authority. Only deterministic Breaker policy may apply the frozen importance
budgets: routine pauses on the first verified evidence-free expansion, important has
at most one falsifiable exploration, and critical requires new verified risk evidence
for every generation. Probe advice cannot change contracts, importance, invariant
identity, failure history, policy, or grants.

Enforce only at a real adapter seam: SDD is `workflow_gate`; approved matching
OpenSpec/Comet revisions are `checkpoint_gate`; generic prompt observations are
`audit_only` and at most warn. Never claim generic real-time tool blocking.

Installation may copy assets, select the versioned runtime, and migrate the selected
schema. It must not add Stop/AfterAgent hooks, import real Guard state, activate Guard
authority, cut over a repository, start a service or scheduler, or create a learning
reader. Real import/cutover, global Skill changes, and runtime canaries require separate
operator authorization.

Use explicit repository identity initialization
(`agent-feedback-loop lineage-init --repo-root "$PWD" --apply`) before the still
read-only Guard dry-run, then import, shadow parity, explicit per-repository cutover,
and exact rollback in that order. Identity initialization creates only the private Git
common-directory lineage; it does not read legacy state, open a Store, or change
authority or hooks. Do not dual-write authority. Independent convergence
effectiveness-to-Markdown learning remains deferred until a named producer, bounded
evidence envelope, and independent learning-job authority/result contract are approved.

Keep structured logs and machine output free of raw prompts, diffs, reviewer/Probe/state
bodies, tokens, grant artifacts, and absolute project paths. Report code verification,
temporary installation, real dry-run, canary cutover, and production effectiveness as
separate evidence states. macOS/Linux target support is not platform acceptance proof.
